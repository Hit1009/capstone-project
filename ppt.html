Introduction

The Problem: Traditional e-learning lacks interactivity. Recorded videos are static, and human tutors are not scalable for 1-on-1 personalized learning.

The Solution: A virtual AI teacher powered by Large Language Models (LLMs) that combines dynamic slides (Reveal.js) with precise Text-to-Speech (TTS).

Core Innovation:

  • Synchronized multimodal teaching (audio matches UI highlights perfectly).
  • Real-time interruption handling (students can pause, ask doubts, and get answers instantly).

Project Objectives

  1. Automated Presentation Engine: Build a scalable frontend using Next.js and Reveal.js to render complex HTML/CSS natively.
  2. Millisecond Audio-Visual Sync: Develop an algorithm to perfectly align TTS playback with visual animations.
  3. Real-Time Doubt Resolution: Implement a robust pipeline where learners can interrupt the session and query the LLM.
  4. Scalable Architecture: Maintain a decoupled Express/Node.js backend and a React frontend.

Presentation State Machine

The system relies on a strict state machine to manage presentation flow:

  • PLAYING: Audio advances continuously; `requestAnimationFrame` polls audio time and triggers `Reveal.nextFragment()`.
  • PAUSED (DOUBT MODE): Audio halts. Current slide index, fragment index, and lesson context are serialized to the LLM.
  • RESUMING: Upon receiving the LLM answer, dynamic TTS plays. Reveal.js UI state is restored precisely.

Data Flow Design

  1. Initialization: Next.js fetches `lesson.ts` from the Express backend via `/api/courses/:id`.
  2. Pre-loading: The browser pre-loads HTML slide content and the corresponding segment audio files.
  3. Synchronized Execution: `useAudioSync` hook maps tags to exact milliseconds, updating DOM classes.
  4. Interruption Pipeline: `DoubtPanel.tsx` triggers a POST to `routes/doubt.ts`, returning a contextual AI response.

Modules Developed & Integrated

  • Presentation Wrapper (`RevealPresentation.tsx`): A React component that safely mounts Reveal.js without SSR conflicts.
  • Audio Engine (`AudioPlayer.tsx`): Manages HTML5 audio contexts, tracking precise playback time and pause/play states.
  • Interactive Doubt Engine (`DoubtPanel.tsx`): UI that captures user queries, communicates with backend, and displays LLM loading states.
  • Backend Routing: Express setup formatted entirely for generating Reveal.js HTML nodes and metadata.

Algorithm 1: Audio-Visual Sync

Goal: Trigger visual CSS highlights exactly when the AI speaks a specific word.

  1. Parsing: Source text has pseudo-tags: "This is the {Highlight: CPU} architecture."
  2. Metadata Mapping: Backend calculates the millisecond offset of "CPU" using Speech Synthesis timestamps.
  3. Execution: Array generated: `[{ timestamp: 4500, targetId: 'CPU', type: 'highlight' }]`.
  4. Polling: React frontend continually checks `audio.currentTime` against this array.

Algorithm 2: Context-Aware Doubts

Goal: Answer student doubts without losing slide context.

  1. Frontend captures the query, `slideIndex`, and `lessonId`.
  2. Backend `routes/doubt.ts` intercepts the request.
  3. Backend searches the active lesson file to extract the exact text the student is viewing.
  4. LLM prompt constructed: "Student is on slide [X]. They asked: [Query]. Answer briefly based on the slide."
  5. Response is sent back for playback.

Code Snippet: Frontend Sync Hook

File: ai-tutoring/hooks/useAudioSync.ts


import { useEffect } from 'react';

export function useAudioSync(audioRef, timingMetadata, reveal) {
    useEffect(() => {
        let animationFrameId;
        const checkSync = () => {
            if (!audioRef.current || !reveal) return;
            
            const currentTime = audioRef.current.currentTime * 1000;
            const nextAction = timingMetadata.find(
                t => !t.triggered && currentTime >= t.timestamp
            );

            if (nextAction) {
                reveal.nextFragment(); // Trigger Reveal.js UI
                nextAction.triggered = true; // Prevent double-firing
            }
            animationFrameId = requestAnimationFrame(checkSync);
        };
        checkSync(); // High-performance loop
        return () => cancelAnimationFrame(animationFrameId);
    }, [timingMetadata, reveal]);
}

Code Snippet: Backend Doubt Pipeline

File: backend/src/routes/doubt.ts


import { Router } from 'express';
const router = Router();

router.post('/', async (req, res) => {
    try {
        const { query, context } = req.body; 
        
        // Connect to LLM with specific Slide Context
        const prompt = `Context: ${context.slideText}. Query: ${query}`;
        const llmResponse = await llmService.generate(prompt);
        
        // Generate Audio for the Response
        const audioUrl = await ttsService.synthesize(llmResponse);

        res.json({ answer: llmResponse, audioUrl });
    } catch (error) {
        res.status(500).json({ error: 'Failed to process doubt' });
    }
});
export default router;

Challenges & Solutions (1/2)

Challenge: Reveal.js vs. React Virtual DOM

Issue: Reveal.js manipulates the DOM directly, while React manages a Virtual DOM. Navigating routing state caused visual crashes and duplicate slides.

Solution: Encapsulated Reveal.js inside a strict React `ref` component. Implemented teardown logic invoking `Reveal.destroy()` upon component unmount to hand full control back to React.

Challenges & Solutions (2/2)

Challenge: Aligning Audio with Slide Actions

Issue: TTS speech rates vary, and browser audio buffering causes hardcoded timestamp delays (`setTimeout`) to rapidly drift out of sync with CSS visual highlights.

Solution: Shifted to an Absolute Time-Tracking Model.

  • Used `requestAnimationFrame` to poll the HTML5 `
  • Built logic to parse specific word-level timestamps generated by the Text-to-Speech synthesis API, ensuring frame-perfect triggers regardless of browser lag.
